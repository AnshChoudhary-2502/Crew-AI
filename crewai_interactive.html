Section 01

The Philosophy

Why CrewAI exists and the thinking behind it

Core idea: Complex problems are solved better by a team of specialists than by a single generalist. CrewAI brings this human truth to AI.
01

Role-Based Thinking

Every agent has a defined identity — a role, goal, and backstory. This focus makes it behave like an expert rather than a generic assistant. A "Senior Data Analyst" agent thinks and responds differently than a "Creative Copywriter" agent, even if both run on the same underlying LLM.

02

Collaborative Autonomy

Agents are not micromanaged step-by-step. You define what needs to be done, and they figure out how — using the ReAct loop (Reason, Act, Observe). They call tools, interpret results, and decide when they're satisfied, all on their own.

03

Composability

Build a crew like LEGO. Snap together different agents, tools, and tasks in different configurations. Reuse the same Researcher agent across multiple projects. Build modular AI pipelines that can be extended without rebuilding from scratch.

04

LLM Agnosticism

CrewAI doesn't lock you into one AI provider. You can use Claude (Anthropic), GPT-4 (OpenAI), Gemini (Google), or even local models like Llama. Each agent can even run on a different model — use a powerful model for the manager and cheaper models for workers.

The Single LLM Problem

When you call a single LLM with a complex task, it must research, analyze, write, review, and format — all in one go. This leads to shallow results, hallucination, and no specialization. CrewAI solves this by letting each agent focus deeply on one job.

Section 02

Core Components

Every part of CrewAI explained in depth

Agent

An Agent is an AI worker with a specific identity. It has a role, a goal, a backstory, an LLM (brain), and tools (abilities). It works autonomously on tasks assigned to it.

role goal backstory llm tools memory verbose allow_delegation max_iter

Analogy: An employee with a job title, a KPI, a résumé, and a set of software tools they know how to use.

Task

A Task is a specific, well-defined piece of work. It has a description (the instruction), an expected output (the format), an assigned agent (who does it), and optionally context from previous tasks.

description expected_output agent context output_file async_execution

Analogy: A work ticket or Jira issue — it says what needs to be done, who does it, and what "done" looks like.

Crew

The Crew is the top-level container. It holds all agents and tasks, and decides how they work together (process). You start everything by calling crew.kickoff().

agents tasks process manager_llm memory verbose

Analogy: The company itself — it has a team, a list of projects, and a management structure (flat or hierarchical).

Tools

Tools give agents real-world capabilities. Without tools, an agent can only generate text. With tools, it can search the web, read files, call APIs, write code, and more.

Built-in
SerperDev, WebScraper, FileRead, CSV, PDF, GitHub, YouTube...
LangChain
100s of integrations: databases, APIs, cloud services, vector stores...
Custom
Any Python function wrapped with @tool decorator becomes a tool
Process

Process defines the workflow pattern — how tasks are ordered and how agents collaborate.

Sequential
Task 1 → Task 2 → Task 3. Output of each task flows into the next. Simple, predictable, easy to debug.
Hierarchical
A manager agent decides task order and delegation. More flexible for open-ended goals.

Section 03

How It Works

The ReAct loop, execution flow, and agent thinking

When you call crew.kickoff(), CrewAI orchestrates all agents and tasks according to the process — automatically passing outputs, calling tools, and managing the ReAct loop for each agent.

You provide a goal

You call crew.kickoff(inputs={'topic': 'AI in healthcare'}). You're giving the crew a starting point — a topic, a question, a dataset, or any inputs your tasks reference.

crew.kickoff(inputs={'topic': 'AI in healthcare 2024'})

Crew decides task order

For a sequential process, tasks run in the order you defined. For hierarchical process, a manager agent reads the overall goal and decides which agent gets which task in what order.

Process.sequential → Task1, Task2, Task3 in order Process.hierarchical → Manager decides dynamically

Agent reasons (ReAct: Reason)

The agent assigned to Task 1 starts. It reads the task description and thinks: "What do I need? Do I need to search the web? Do I need to read a file? Can I answer from my own knowledge?"

Agent thinks: "I need to search for recent AI healthcare news. I should use the SerperDevTool to find current articles."

Agent acts (ReAct: Act)

If the agent decides it needs a tool, it calls it automatically. CrewAI handles the tool call, waits for the result, and feeds it back to the agent. No manual code needed — the agent figures out which tool to use.

→ Calling SerperDevTool("AI in healthcare 2024") → Tool returns: 10 search result snippets...

Agent observes (ReAct: Observe)

The agent reads the tool result and decides: "Is this enough? Do I need to search again? Do I need to fetch a specific article?" It can loop — call more tools, refine queries — until it's satisfied. Max iterations prevent infinite loops.

Agent: "I found 8 relevant articles. This is sufficient. Now I will synthesize a summary..."

Agent produces output

When satisfied, the agent writes its final answer as the task output. This output matches the expected_output format you defined in the task. The task is complete.

Task 1 output: "Top 10 AI healthcare trends: 1. AI diagnostics growing 40% YoY... 2. LLMs entering clinical workflows..."

Output flows to the next task

Task 1's output is automatically passed as context to Task 2. The next agent reads it and builds on it. This chain continues until all tasks complete. The final task's output is what kickoff() returns to you.

Task 2 (Analyst) receives Task 1's research as context → Identifies 5 key trends → Task 3 (Writer) receives those trends → Writes final 800-word article ✓

Section 04

Code Guide

Real Python code with explanations for every part

1. Installation
pip install crewai crewai-tools
2. Creating an Agent
from crewai import Agent from crewai_tools import SerperDevTool # The search tool gives the agent web access search = SerperDevTool() researcher = Agent( role="Senior AI Researcher", # Identity / persona goal="Find accurate, current AI trends", # What it wants to achieve backstory="You are a PhD researcher at a leading AI lab. \ You only cite peer-reviewed sources.", # Context that shapes behavior tools=[search], # What it can DO verbose=True, # Show thinking steps allow_delegation=False, # Can it assign sub-tasks? max_iter=5, # Max ReAct loop iterations )
3. Creating a Task
from crewai import Task research_task = Task( description="""Search for the top 10 AI developments in healthcare from the past 6 months. Focus on clinical trials, FDA approvals, and major research papers.""", expected_output="""A numbered list of 10 AI healthcare developments. For each: title, date, source, 2-sentence summary.""", agent=researcher, # Which agent does this output_file="research.md" # Save output to a file (optional) )
4. Chaining Tasks with Context
writer = Agent( role="Medical Content Writer", goal="Write clear, engaging articles for a general audience", backstory="You write for patients, not doctors. Simple language only." ) writing_task = Task( description="Write a 600-word article summarizing the key findings.", expected_output="A complete article with intro, body, conclusion.", agent=writer, context=[research_task], # ← Gets research_task's output automatically output_file="article.md" )
5. Assembling the Crew
from crewai import Crew, Process crew = Crew( agents=[researcher, writer], tasks=[research_task, writing_task], process=Process.sequential, # or Process.hierarchical memory=True, # Enable long-term memory verbose=True ) # Fire it up! result = crew.kickoff(inputs={'topic': 'AI in healthcare 2024'}) print(result)
6. Custom Tool (Advanced)
from crewai.tools import tool # Any Python function can become a tool with @tool @tool("Stock Price Checker") def get_stock_price(ticker: str) -> str: """Fetches the current stock price for a given ticker symbol.""" # Your actual logic here return f"AAPL is currently $182.50" # Now pass it to any agent analyst = Agent( role="Financial Analyst", goal="Analyze stock performance", backstory="Expert in equities and market trends", tools=[get_stock_price] )

Section 06

CrewAI vs Others

How CrewAI stacks up against similar frameworks

CrewAI ✦

  • Role-based agents with backstory
  • Sequential & hierarchical processes
  • Built-in memory system
  • Very easy to set up
  • Great for teams of 2–10 agents
  • Human-in-the-loop support
  • LangChain tool compatibility

LangChain Agents

  • Single-agent focused
  • No built-in multi-agent orchestration
  • Highly customizable
  • More complex to set up
  • Massive ecosystem of tools
  • Lower-level control
  • CrewAI uses LangChain tools

AutoGen (Microsoft)

  • Conversation-based multi-agent
  • Agents talk to each other naturally
  • Better for code generation tasks
  • More complex configuration
  • Less structure on roles/goals
  • Strong Microsoft ecosystem

LangGraph

  • Graph-based agent workflows
  • Very flexible state management
  • Steeper learning curve
  • Best for complex branching flows
  • Fine-grained control over state
  • Built by LangChain team
When to choose CrewAI: You have a workflow that maps naturally to human team roles. You want fast setup with minimal boilerplate. You're building pipelines like research → analyze → write → review. You want role specialization and don't need extremely complex branching logic.
